Implementa la generación automática y correcta de dieciseisavos proyectados del Mundial 2026 a partir de las apuestas de fase de grupos de cada usuario.

Contexto:
La app es una porra del Mundial 2026 en Streamlit con PostgreSQL.
La base ya existe y NO debe modificarse.
Ya existen:
- players
- teams
- matches
- predictions
- app_settings
- pages/01_Apuestas.py

La tabla matches contiene:
- 72 partidos de fase de grupos con stage='group'
- 16 placeholders de dieciseisavos con stage='round_of_32' y match_number 73 a 88

Archivos que ya existirán antes de ejecutar este prompt:
- data/third_place_assignment_2026.json
Este archivo contiene las 495 combinaciones oficiales de asignación de terceros.
Debe tener esta estructura:
{
  \"_metadata\": {...},
  \"assignments\": {
    \"ABCDEFGH\": {
      \"1A\": \"3H\",
      \"1B\": \"3G\",
      \"1D\": \"3B\",
      \"1E\": \"3C\",
      \"1G\": \"3A\",
      \"1I\": \"3F\",
      \"1K\": \"3D\",
      \"1L\": \"3E\"
    }
  }
}

IMPORTANTE:
- No modificar el schema de base de datos.
- No borrar datos.
- No hacer DROP.
- No cambiar reglas de scoring existentes.
- No modificar los matches globales 73-88 con equipos concretos, porque el bracket proyectado depende de cada usuario.
- La lógica debe ser por usuario.
- No inventar asignaciones de terceros: usar exclusivamente data/third_place_assignment_2026.json.
- Si falta el JSON o no contiene 495 combinaciones, mostrar error claro y no generar cruces incompletos.

Objetivo:
Cuando un usuario haya apostado los 72 partidos de fase de grupos, la app debe calcular automáticamente:
1. La clasificación proyectada de cada grupo.
2. Los 8 mejores terceros proyectados.
3. La clave de terceros, por ejemplo ABCDEFGH.
4. Los cruces de dieciseisavos usando data/third_place_assignment_2026.json.
5. Permitir apostar marcador y equipo que avanza en esos cruces proyectados.

Crear:
- bracket.py
- tests/test_bracket.py si el proyecto ya usa pytest o si es fácil añadirlo.
- Opcionalmente utils/bracket_ui.py si ayuda a mantener limpia la UI.

En bracket.py implementar funciones puras:

     1A, 1B, 1D, 1E, 1G, 1I, 1K, 1L.
   - Si falla, devolver error controlado o lanzar ValueError con mensaje claro.

2. get_player_group_predictions(session, player_id)

   - Debe devolver apuestas con:
     match_id
     match_number
     group_letter
     home_team_id
     home_team_name
     away_team_id
     away_team_name
     predicted_home_score
     predicted_away_score
   - Solo considerar partidos con ambos equipos definidos.

3. calculate_projected_group_tables(group_predictions)
   - Calcula tabla por grupo con:
     team_id
     team_name
     group_letter
     played
     won
     drawn
     lost
     goals_for
     goals_against
     goal_difference
     points
   - Victoria = 3 puntos.
   - Empate = 1 punto.
   - Derrota = 0 puntos.

4. rank_group_table(table)

     Match 81: 1D vs tercer asignado para 1D
     Match 82: 1G vs tercer asignado para 1G

   - Devolver lista con:
     match_number
     home_slot
     away_slot
     home_team_id
     home_team_name
     away_team_id
     away_team_name

8. get_projected_bracket_for_player(session, player_id)
   - Orquestar todo.
